# Go工作流 验证结果

验证日期: 2026-06-12

## 验证范围

本次验证基于当前文件夹内的 `go_workflow.py`，未修改 Blender 用户配置、偏好设置或已安装插件目录。

验证命令使用 Blender 4.2.9 的 `--factory-startup` 临时加载当前文件夹源码。

## 已实现并通过验证

- 工作流新增、删除、重命名、上下排序。
- 默认工作流显示全部第三方 `VIEW_3D > UI` N 面板。
- 非默认工作流按勾选结果隐藏其他第三方 N 面板。
- 隐藏方式为临时 `unregister_class` 未选中的第三方 Panel 类，切回默认工作流会恢复。
- 父子面板联动: 勾选父面板时，子面板会随父面板保留。
- 面板库扫描、刷新、缺失面板红色提示。
- 面板显式加入/移除工作流。
- 当前工作流面板一键全部加入、一键清空、上下排序。
- 面板标签配置。
- 工作流“按标签自动加入”配置。
- 工作流通过标签自动加入命中的面板。
- 工作流说明保存，并随预设导出导入。
- `.bworkflow` 预设导出导入，包含工作流、面板、标签、模块、模块代码内容和 AI 辅助说明。
- 独立设置 UI: 工作流、面板库、脚本模板、预设。
- 工作流直接操作 UI: 工作流切换、自定义脚本模块运行。
- 自定义模块新增、删除、排序、启用/禁用。
- 自定义模块固定入口协议: `run(context, scene, workflow, module)`。
- 自定义模块 AI 开发文档生成和复制。
- 自定义模块 `.py` 模板写入当前插件文件夹下的 `go_workflow_modules`，不会写入 Blender 用户脚本目录。
- 调试输出当前扫描到的第三方 N 面板清单。

## 替代实现或限制

- 真正的并列“双层 N 面板区域”无法通过 Blender Python API 创建。当前实现采用两个 N 面板标签: `工作流` 和 `独立设置`。
- 鼠标拖拽排序暂未实现，当前用上下按钮排序。
- 批量折叠/展开第三方面板暂未实现，Blender Python 对原生面板折叠状态控制有限。
- 全局 UI 样式设置如宽度、字体大小、警示颜色、拖拽灵敏度暂未实现。
- 临时注销第三方 Panel 类可以让 N 标签消失，但属于更强的运行时控制。插件卸载和切回默认工作流会恢复已注销面板。
