# 表情预设开发方案

## 目标

在 Go工作流 插件中增加一个“表情 / ARKit 52 形态键参考预设”。

该预设的核心能力：

1. 选择一个参考步骤。
2. 自动把对应的形态键设置为 `1.0`。
3. 自动把其它 ARKit 参考形态键恢复为 `0.0`。
4. 自动切换到编辑模式。
5. 弹出参考图窗口，供当前步骤对照。
6. 在面板里显示该步骤的注意重点与小技巧。

## 推荐实现方式

### 1. 数据层

新增一份内置 JSON 数据文件，例如：

- `go_workflow/special_presets/arkit形态键工作流参考.json`

每个步骤记录：

- `id`
- `shape_key`
- `name_en`
- `name_zh`
- `name_bilingual`
- `category`
- `summary`
- `tips`
- `notes`
- `image_hint`
- `image_local_path`
- `source`

其中：

- 与 Blender 形态键直接对应的项目，名称保留双语。
- `shape_key` 必须与 Blender 中的形态键名称一一对应，便于直接驱动。

### 2. UI 层

建议增加一个脚本模块或一个内置特殊面板：

- 面板名建议：`arkit形态键工作流参考`

建议提供：

1. 目标物体选择器
2. 步骤下拉选择
3. 上一步 / 下一步
4. 应用当前步骤
5. 重置全部参考形态键
6. 打开参考图
7. 自动进入编辑模式开关

### 3. 运行逻辑

点击“应用当前步骤”时：

1. 校验目标物体存在。
2. 校验目标物体有 `shape_keys`。
3. 校验当前步骤对应的 `shape_key` 存在。
4. 将 ARKit 参考列表中的所有形态键值设为 `0.0`。
5. 将当前步骤的形态键值设为 `1.0`。
6. 如果开启“自动进入编辑模式”，则切换到编辑模式。
7. 记录最近一次步骤状态。

### 4. 参考图显示

最稳妥的方式不是自定义 OS 级弹窗，而是：

1. 将参考图下载到本地。
2. 用 Blender 的 `Image Editor` 显示该图片。
3. 如果当前窗口没有 `IMAGE_EDITOR`，则：
   - 优先尝试把现有区域临时切到 `IMAGE_EDITOR`
   - 或打开新窗口显示参考图

这样比系统弹窗更稳，也更容易跨平台。

### 5. 注意重点与技巧显示

当前步骤选中后，在面板中显示：

- 中文说明
- 英文原名
- 注意重点
- 小技巧

建议显示为短句，不要整段长文本。

## 更适合你当前插件的接法

建议不要把它做成“普通工作流预设导入导出”的一部分，而是做成：

1. 内置参考数据
2. 专用脚本模块模板
3. 可选的一键创建工作流

推荐流程：

1. 在插件内提供“创建 arkit形态键工作流参考”按钮。
2. 自动创建一个带专用模块的工作流。
3. 模块读取 `go_workflow/special_presets/arkit形态键工作流参考.json`。
4. 在运行面板里显示步骤切换与参考信息。

## 为什么这样更稳

原因：

1. 不依赖外部网页在线访问。
2. 不依赖用户手动下载图片。
3. 不污染通用工作流预设结构。
4. 后续维护时只需要改 JSON 和图片目录。
5. 可以继续沿用你现在的 `run(...) + draw_panel(...)` 模块体系。

## 下一步建议

1. 生成 `go_workflow/special_presets/arkit形态键工作流参考.json`
2. 生成一个可直接放进脚本模块库的 Blender Python 模块
3. 给 Go工作流 增加“一键创建 ARKit 参考工作流”按钮
