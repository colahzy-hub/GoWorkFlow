# Blender Go工作流插件需求规格说明书

## 1. 产品定位
本插件是 Blender 的工作流与 N 面板管理工具，用于按工作流管理第三方面板显示、顺序、脚本模块与预设导入导出。

## 2. 术语定义
- 工作流：一个独立配置单元，保存面板勾选、显示顺序、脚本模块与说明。
- 面板组：文中默认指“抽屉组”，即按抽屉根面板归一后的分组。
- 抽屉：第三方面板中可折叠显示的一组面板；同一抽屉可能包含多个子面板。
- 子抽屉：一个抽屉组内部的实际面板成员，用于排序、显示和计数。
- 抽屉计数：一个面板组里实际展示的唯一抽屉条目数量，不等于子面板总数。

## 3. 核心目标
- 保留 Blender 原生面板机制，不修改 Blender 内置 UI 注册代码。
- 通过插件独立管理第三方面板的显示、隐藏与顺序。
- 通过工作流组织不同场景的面板集合与脚本模块。
- 支持预设导入导出与跨机器共享。

## 4. 版本与打包
- 当前实现版本：`0.4.12`
- 版本快照保存在 `version_history/`
- 预设文件格式：`.goworkflow`
- 预设导入需兼容旧格式：`.bworkflow`
- 插件发布包使用 `version_history` 内的 `zip`
- 发布包根目录保留 Blender 扩展结构，至少包含：
  - `go_workflow/__init__.py`
  - `go_workflow/blender_manifest.toml`
- 当代码拆分为多文件后，发布包必须同时包含 `go_workflow` 目录下的全部运行所需 `.py` 文件，不允许只打包单个 `__init__.py`

## 5. 当前已实现功能
### 5.1 工作流管理
- 新建、复制、删除、重命名工作流
- 设置默认工作流
- 调整工作流顺序
- 清空当前工作流面板
- 通过标签自动加入面板

### 5.2 面板管理
- 面板库扫描、刷新、缺失提示
- 当前工作流面板的勾选、加入、移除
- 父子面板联动保留
- 当前工作流面板组顺序重置、反转、移动
- N 面板显示/隐藏与顺序刷新
- 主插件 UI 中的工作流说明、模块说明、脚本内容预览、AI 文档预览和帮助说明默认折叠；展开后才绘制多行文本，避免长文本导致 N 面板卡顿

### 5.3 脚本模块
- 脚本模块新增、删除、排序、启用/禁用
- 模块支持自定义面板
- 模块支持 `run(context, scene, workflow, module)` 入口
- 可生成 AI 开发说明文本

### 5.4 脚本库
- 当前模块可保存到脚本库
- 脚本库条目可载入到当前模块
- 可按工作流切换脚本库条目启用状态
- 脚本库内容以源码与元信息保存，避免跨工作流互相覆盖
- 脚本库源码刷新优先读取 Blender 文本块，其次读取脚本文件
- 内置脚本库条目允许通过缓存减少重复读盘，避免主界面高频卡顿

### 5.5 预设
- 导出勾选的工作流为 `.goworkflow`
- 导入 `.goworkflow` 预设并恢复工作流、面板、脚本模块
- 载入预设后可多选再导入
- 导入时保留缺失面板记录与警告信息
- 预设导出默认只携带当前导出工作流相关的脚本库项，减少体积并提升导入导出速度
- 内置脚本条目与内置特殊预设应尽量依靠安装包内资源重建，不重复塞进预设

### 5.6 AI 文档
- 每个脚本模块可生成 AI 开发说明文本
- AI 文档需要包含可直接识别的基础脚本模板骨架：`import bpy`、`_selected`、`_validate`、`run(context, scene, workflow, module)`
- 开启自定义面板时，AI 文档必须额外包含 `draw_panel(...)` 与 `on_panel_action(...)` 的 UI 骨架
- 面板 UI 骨架应使用 `panel_api.section`、输入控件、状态显示、`draw_button`、`draw_run_button`，字段 key 使用英文，label 可用中文
- AI 文档需要控制长度，优先保留模块目标、接口约束、基础骨架、自定义面板开关和关键 API 提示
- AI 文档应避免塞入完整业务实现；长说明放到模块说明或外部文档，模板只保留最小可维护骨架

### 5.7 ARKit 形态键合成表
- ARKit 形态键合成的混合系数表按“目标预设 × 混合率”独立保存
- 目标预设至少区分 `VRM 基础` 与 `MMD 部分`
- 混合率至少区分 `常规` 与 `激进`
- 因此最少应分别存在四套表：`VRM-常规`、`VRM-激进`、`MMD-常规`、`MMD-激进`
- 当前面板选中哪一套，导入、导出、校验、清空就只作用于这一套，不影响其他三套
- 内置默认表与额外导入自定义表都必须遵循同样的“四套独立”逻辑
- 用户可通过 Blender 原生文件浏览器把当前选中的表导出为 `csv` 或 `txt`
- 用户修改导出的 `csv/txt` 后，可再次通过 Blender 原生文件浏览器重新导入到当前选中的表
- 若当前选中的是内置默认表，导出时应直接导出当前这一套默认表，而不是导出别的目标预设或别的混合率
- 若当前选中的是自定义表，清空操作只清空当前这一套自定义表，并恢复当前这一套的内置默认表

## 6. 面板组与抽屉排序规则
### 6.1 面板库左侧排序
- 面板库按抽屉根面板归类。
- 同一组内先去重，再按名称排序。
- 组顺序按组标题排序，标题相同时按分组键排序。
- 组内条目按面板标题排序，标题相同时按 `panel_id` 排序。
- 一个组如果只有 1 个抽屉且当前工作流未选中，可不在库列表中显示。

### 6.2 当前工作流右侧排序
- 当前工作流右侧面板组顺序以工作流内第一次出现的顺序为准。
- 同一组内的条目必须保持在同一组里，不能被拆散。
- 组顺序调整只影响组之间的相对顺序，不应破坏组内成员关系。
- 组内条目排序以当前工作流中的出现顺序为主，其次按标题和 `panel_id` 排序。

### 6.3 子抽屉排序
- 子抽屉排序是同一面板组内部成员的排序。
- “子抽屉默认排序”必须按插件扫描到的默认顺序整理。
- 子抽屉排序后，Blender N 面板中的实际注册顺序也必须同步变化。
- 子抽屉排序不能改成按面板组数量排序，也不能改成按父面板数量排序。
- 如果某个抽屉是父子结构，排序时必须保证父面板在前、子面板在后。

## 7. 抽屉计数规则
- 面板库中的计数按“抽屉条目数”统计。
- 计数只统计该组里唯一的抽屉根条目，不统计子面板重复项。
- 当前工作流右侧的组计数按该组 `entries` 数量统计。
- 计数不应改成“组内所有子面板数量”，也不应改成“展开后的总层级数量”。
- 如果组内条目去重后只有 1 个，则计数应显示为 1。
- 计数规则以当前实现和 `0.3.16` 的抽屉计数口径为准，后续不得随意改成其他统计口径。

## 8. 需求约束
- 面板计数必须保持为抽屉计数，不得被改成子抽屉总量、树深计数或其他口径。
- 隐藏功能必须真正作用于 Blender N 面板。
- 顺序功能必须真正应用到 Blender N 面板注册顺序。
- 不删除用户数据，删除操作只影响当前配置。
- 不依赖实时扫描整个插件库的高频重操作。

## 9. 验收标准
- 面板库左侧组数、组标题、组内条目数与实际抽屉一致。
- 右侧工作流组顺序变更后，N 面板顺序能同步变化。
- 子抽屉默认排序执行后，N 面板实际顺序与插件默认扫描顺序一致。
- 面板隐藏后切换工作流，隐藏状态能恢复或保持符合当前工作流配置。
- 预设导入导出均使用 `.goworkflow`，并保留缺失面板警告。
- 发布包 `zip` 必须可直接安装，并包含 `go_workflow` 目录下全部必要脚本文件与 `blender_manifest.toml`。

## 10. 说明
本文件以当前代码实现为准；后续如修改行为，应同步更新本需求文档与 `version_history` 中对应快照。
